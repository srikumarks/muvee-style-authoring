3 Style Package Structure

muvee Reveal comes bundled with several styles that you can use to personalize your muvees. More styles are also available from our style labs page.

This document describes the package structure of all styles, including those shipped via our Style Labs page, but the muveeStyleBrowser provides a more convenient way to access, download, install, uninstall and edit the styles published on muvee-style-authoring. You don’t need to get into these details if you do not want to customize any of muvee’s own commercial styles, including those bundled with muvee Reveal.

You can use this information to -

  • Change the name of a style,

  • Localize a style’s name and description to your taste,

  • Change a style’s icon,

  • Change any stock graphics used by the style,

  • Create variations of a style.

3.1 Location

Styles are usually installed in the directory - "C:\Program Files\Common Files\muvee Technologies\071203\". The drive letter might differ depending on your installation location. A standard muvee Reveal installation will have the following style packages in the above installation directory -
S00509_MusicVideo3
S00510_SportHex
S00511_Journal2
S00512_Strips
S00513_Reflections2
S00514_UltraPlain
S00516_Plain
S00517_Showreel
S00518_Cube

The name of a style package, as exemplified above, has the structure -

"S<5-digit-number>_Identifier"

A style package is a directory with specific structure and contents.

3.2 Package contents

Let us look at the “Ultra Plain” style’s package structure, which is the simplest of the bundled styles.

  • S00514_UltraPlain/

    • icon.png

    • preview.wmv

    • strings.txt

    • data.scm

Every style must have at least these four files in order to be visible within the muvee Reveal interface and these files must have exactly the names shown above.

3.2.1 icon.png

This is the style’s icon which will be displayed in the styles pane of muvee Reveal. The format is as follows -

  • Dimensions = 50w x 50h

  • Format = png

  • Bit depth = 32

3.2.2 preview.wmv

This is a video file that shows what the style looks like. It is played in the muvee Reveal interface when you select the style. You can generate a preview video for your own styles using muvee Reveal itself. The format of the preview file is as follows -

  • Dimensions = 240w x 180h

  • Duration = 10 seconds

  • Format = WMV9

  • Bit rate approximately 140kbps

  • No sound track

3.2.3 strings.txt

A UTF-8 encoded plain text file that contains the localized name and description strings. It also contains the localized names of all the parameters exposed by the style in its Style Settings panel. You can view the contents of this file using the Notepad program that comes with Windows XP or Vista.

3.2.4 data.scm

A plain text file (utf-8) containing a script that specifies the various aspects of a style such as exposed parameters, music response, effects and transitions, title/credits and such. For more on the contents of a "data.scm" file, see Specifying Style Behaviour.

3.3 Resources

The “Ultra Plain” style is .. well .. ultra plain, but other styles such as “Hexplode” (S00510_SportHex) and “Scrapbook” (S00511_Journal2) make use of images and video clips to create their unique looks. The folders of these styles therefore also contain additional image and video files.

There is no general naming convention used for style resources and you might find that some of them are obvious (ex: background.wmv) whereas others aren’t (ex: 004.png). The role played by a particular resource file is specific to the style.

If you change a style’s graphics resources, while retaining the original file names, you can dramatically change the look of a style. You can also mix and match resources from different styles to create hybrid styles. If you’re a Photoshop-er, you might have fun playing around with these files.

3.4 Creating a variant style

Now that you know about the structure of a style, it is time to create your own variant of an existing style.

  • Select the package folder of the style that you want to create a variant of.

  • Copy the style package in its entirety and give it a unique name along the lines of "S<5-digit-number>_Identifier". You can copy, say, "S00517_Showreel" and name the folder as "S00517_MyShowreel".

  • Change the name of your style.

    • Open the "strings.txt" file in your copy,

    • Locate the line that says

      STYLENAME    en-US    Uncle Oscar

      (for English text)

    • Edit the “Uncle Oscar” part to whatever you want to call your variant, say “Film strips”.

    • (Re)launch muvee Reveal. You’ll find a new style in the panel called “Film Strips”.

You can then customize the variant style’s icon, preview and graphics resources to achieve the different look that you desire.

If you think you’ve created something interesting, do write about it to the muvee-style-authoring group.

3.5 More about strings.txt

Open a "strings.txt" file using Notepad and you’ll see that it consists of a 3-column table with a single <tab> character separating the columns. The first column gives the String ID.

The ID STYLENAME stands for the name of a style as displayed in the muvee Reveal interface. The ID STYLEDESC stands for the descriptive string shown along with the name of the style in the muvee Reveal interface. These two IDs are present in the strings.txt file of any style.

The second column selects a language. For example, "en-US" stands for "English US". In conjunction with the string ID of the first column, it uniquely selects a unicode string for use in the muvee Reveal interface. For example, STYLENAME en-US selects the string for the style’s name in "English US", the entry STYLEDESC ru-RU selects the string for the style’s description in "Russian", and so on.

The third column contains the actual string encoded in UTF-8.
